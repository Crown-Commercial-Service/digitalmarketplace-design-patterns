Help a supplier know their application has been submitted

This design helps a supplier know they’ve successfully submitted an application.

How the design works

Use this design to help suppliers be clear when they’ve submitted their framework application successfully.

Currently, to submit an application a supplier has to:

  • complete or confirm their company details
  • create a supplier declaration
  • add details of at least one service

Once these tasks have been completed their application will be submitted on the deadline.

Suppliers can keeping adding more services right up to the framework application deadline.

Why this is useful

Suppliers need to know that they’ve completed all the necessary steps to apply and have submitted their application on time.

Things to consider

This design needs to be thought about in more detail.

There’s no ‘submit application’ button which has confused suppliers. It makes suppliers question if their application will be submitted. Suppliers regularly raise clarification questions querying this. The only confirmation suppliers receive comes after the application deadline.

It would also be worth considering:

  • emailing suppliers who’ve started an application before the deadline to let them know if their application will be submitted
  • making it clearer that if you don’t mark services as complete you haven’t submitted an application
  • allowing users to submit an application before the deadline and confirming it’s happened
