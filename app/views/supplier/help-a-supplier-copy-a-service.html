Help a supplier copy a service

This design helps a supplier to copy and reuse services they are already offering.

How the design works

Use this design if you need to:

  • reuse existing services
  • allow users to copy services

This design allows suppliers to copy all the information from any services they submitted for a previous framework application.

This makes it quicker and easier for anyone who wants to move from one iteration of a framework to the next.

Copying a service is straightforward.

  • Suppliers click the ‘copy’ button next to the name of an existing service.
  • This creates an exact copy of the service and puts it into ‘draft’.
  • The new service has the word ‘draft’ at the end of the title.
  • Suppliers can then edit the draft service if required.

Suppliers can choose to copy all of their services or one by one.

A confirmation banner confirms when each draft copy of a service has been created.

Why this is useful

Creating a new service involves answering a lot of questions.

Allowing suppliers to copy any existing service, and editing them if necessary, saves a lot of time and effort.

Things to consider

During the application process for G-Cloud 10, over 50% of suppliers used the copy services feature. For applications to Digital Outcomes and Specialists 3, this number was around 30%. We think this is because Digital Outcomes and Specialists only requires suppliers to add one service per lot so the benefits of copying answers were not as great.

However, this design was developed very quickly and there was no time to do any user research.

It might be worth considering that:

  • the ‘copy all services’ link disappears when you click it which may be an issue if suppliers want to copy services multiple times
  • the ‘copy all services’ link should be more prominent on the page
