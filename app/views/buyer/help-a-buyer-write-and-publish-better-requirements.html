Help a buyer write and publish better requirements

This design helps buyers write requirements with all the information suppliers need to apply for the work.

How the design works

This design:

  • presents the buyer with a series of tasks to complete to create their requirements
  • asks a buyer to answer a number of questions
  • publishes the buyer’s responses as their requirements
  • shows the user which steps they’ve already done
  • makes sure all the relevant information is presented to the supplier in the published requirements

Components used in the pattern

This design is made up of the:

  • task list pattern
  • check your answers pattern

Why this is useful

This design:

  • gives buyers confidence that they’ve given suppliers all the information they need to apply
  • means requirements are easier for suppliers to understand
  • makes sure buyers are following the rules of the framework by making them provide all the required information, for example setting evaluation criteria upfront
  • helps members of the public see what government is buying because it’s publicly available

Things to consider

Don’t use this design for a direct award framework.

What we did on the Digital Marketplace

This design allows suppliers to view the available opportunities to apply for work in one place and in a consistent format. It also means all available work is published publicly.

What we learnt

Through testing and using this pattern we learnt that:

  • the lack of a collaborative workflow meant buyers drafted requirements with colleagues using email and shared logins
  • business and procurement teams work in isolation meaning requirements can lack detail or quality leading to poor responses
  • buyers wanted to see a preview of their answers as requirements can’t be edited once published

How to improve this

Still worth considering:

  • that there are a lot of questions to answer, which makes a lot of work for buyers
  • making it clearer when steps in the buying process are locked, for example by using the narrow a search design
  • helping buyers to improve the quality of the content they’re providing
  • providing a quality control function on the content buyers provide
  • improving collaboration online, for example by giving several users access to a shared buyer account
