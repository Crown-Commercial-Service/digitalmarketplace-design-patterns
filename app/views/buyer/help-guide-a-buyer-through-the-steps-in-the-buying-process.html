Help guide a buyer through the steps in the buying process

Use this design if you need to guide users through the buying process in a specific order.

This design is based on the task list pattern in the GOV.UK design system.

How the design works

This design gives users a:

  • list of tasks
  • description of each task, with guidance in some cases
  • ‘Complete’ label once each task is finished
  • set order to do tasks in

Why this is useful

Users who are buying goods and services from a framework need to:

  • follow the procurement regulations/buy fairly and openly
  • know what they’re doing is right
  • create an audit trail
  • find the product that best meets their needs

Things to consider

Don’t use this if users can complete tasks in any order.

There’s more discussion about the standard task list design pattern in the design system’s community pages.

What we did on the Digital Marketplace

A task list could be used for any framework. In this example, it's used on the G-Cloud start page.

For G-Cloud, we needed users to follow 5 steps to buy. Each step needs to be completed in order.

What problem were we trying to solve

We wanted to:

  • learn how buyers would respond to a more guided online G-Cloud journey
  • understand how we could create a journey for G-Cloud buyers which would make capturing award data easier

What we learnt

We looked at how users were buying using G-Cloud.

We found:

  • there are 2 main use cases – people who need to do some research and people who need to buy a service through G-Cloud
  • users with in-house procurement software stop using the Digital Marketplace once they’ve downloaded or copied their search results and use their own systems to complete the buying process

How to improve this

More work needs to be done to think about:

  • why returning to the task list doesn’t seem to be a natural step for buyers after they’ve downloaded their search results
  • giving more people access to the buyer account so they can work together, for example when shortlisting and evaluating
  • how to help buyers after they’ve exported their search results, for example they have to evaluate suppliers offline
  • why the ‘I have read and understood’ button (on step 3) isn’t what buyers expected to see
  • why some buyers don’t know about or won’t complete the customer benefits record

Where else the pattern’s used

On the Digital Marketplace, we use the pattern in 3 places:

  • the G-Cloud start page
  • the Digital Outcomes and Specialists requirements page
  • when suppliers are applying to a framework
